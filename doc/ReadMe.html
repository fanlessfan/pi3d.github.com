Introduction to Pi3D¶

Pi3D written by Tim Skillman, Paddy Gaunt, Tom Swirly Copyright (c) 2013

There’s plenty of 3D code flying around at the moment for the Raspberry Pi, but much of it is rather complicated to understand and most of it can sit under the bonnet!

pi3d is a Python module that aims to greatly simplify writing 3D in Python whilst giving access to the power of the Raspberry Pi GPU. It enables both 3D and 2D rendering and aims to provide a host of exciting commands to load in textured/animated models, create fractal landscapes, shaders and much more.

This is the third release of the pi3d module which has various demos of built-in shapes, landscapes, model loading, walk-about camera and much more! See the demos included with this code and experiment with them ..

Demo’s included with Pi3D¶

  1. ForestWalk.py Walk about a forest on a landscape generated from a bitmap
  1. Triceratops.py Large model loading with several bitmaps
  1. BuckfastAbbey.py Explore a model of the beautiful Buckfast Abbey in Buckfastleigh, Devon
  1. Earth.py Demonstrates semi-transparent clouds and hierarchical rotations
  1. Raspberry_Rain.py Raining Raspberries, full-screen, over the desktop
  1. Clouds3D.py Blended sprites in perspective view
  1. CollisionBalls.py More bouncing balls across the screen - this time bouncing off each other on the desktop
  1. EnvironmentCube.py New environment cubes to try out in texture/ecubes - some high quality ones!
  1. RobotWalkabout.py Another off-planet example of a basic avatar robot drifting about
  1. Shapes.py Demos available shapes and text in a 3D context

11) MarsStation.py Navigate around an abandoned Mars base-station with open/shut doors. Implements a new Level-Of-Detail (LOD) feature and TKwindow interface

12) TigerTank.py Ever played World Of Tanks (WOT)? This tank emulates how a WOT tank works. Uses realistic modelling and LODing in a TKwindow

  1. Amazong.py Can you find yourself around the amazing maze? Written by Paddy Gaunt.
  1. Pong.py A snazzy 3D version of landscape pinball and pong against a Raspberry! Written by Paddy Gaunt.

Setup on the Raspberry Pi¶

  1. Memory Split setup

Although most demos work on 64MB of memory, you are strongly advised to have a 128MB of graphics memory split, especially for full-screen 3D graphics. In the latest Raspbian build you need to edit the config.txt file (in the boot directory) and set the variable ‘gpu_mem=128’ for 128MB of graphics memory.

  1. Install Python Imaging

Before trying any of the demos or Pi3D, you must download the Python Imaging Library as this is needed for importing any graphics used by Pi3. To install on the terminal, type:

sudo apt-get install python-imaging
  1. Install Geany to run Pi3D

Geany is by far the easiest and most compatible application to use for creating and running Python scripts. Download and install it with:

sudo apt-get install geany xterm
  1. Optionally, install tk.

Some of the demos require the tk graphics toolkit. To download and install it:

sudo apt-get install tk
  1. Load and run

Load any of the demos into Geany and run (using the cogs icon)
